# CompeteHub 演示流程清单

## 启动

1. 后端终端：

   ```powershell
   .\scripts\start_backend.ps1 -Seed
   ```

2. 前端终端：

   ```powershell
   .\scripts\start_frontend.ps1
   ```

3. 打开：

   ```text
   http://localhost:5173
   ```

## 演示账号

- 学生账号：`student / student123`
- 管理员账号：`admin / admin123`

## 学生主链路

1. 打开登录页，使用学生账号登录。
2. 进入赛事查询页，确认赛事列表能加载。
3. 打开任意赛事详情页。
4. 点击收藏，页面提示收藏成功。
5. 点击订阅提醒，页面提示订阅成功。
6. 进入个性化推荐页，确认推荐结果和推荐理由能展示。
7. 进入订阅日历页，确认刚订阅的赛事出现在列表中。

## 未登录行为

1. 退出登录或清空登录状态。
2. 在赛事列表或详情页点击订阅/收藏。
3. 页面应提示请先登录，并跳转到登录页。
4. 登录成功后应回到原页面。

## 管理员主链路

1. 使用 `admin / admin123` 登录。
2. 打开后台管理页。
3. 确认用户数、赛事数、收藏数、订阅数、论坛帖子数等统计能加载。

## 当前阶段说明

- 本地演示默认使用 SQLite：`backend/instance/dev.db`。
- PostgreSQL 仍保留在正式部署方案中，后续可通过 Docker 或服务器环境切换。
- 阶段一重点是稳定演示闭环；赛事新增、审核、编辑等后台管理能力放在下一阶段继续补齐。
