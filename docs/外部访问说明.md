# CompeteHub 访问说明

这份文档说明如何在本机打开项目，以及如何让同一个 WiFi/局域网下的其他人访问你的页面。

## 一、先说结论

你当前电脑的 WLAN 局域网 IP 是：

```text
10.90.11.95
```

所以，在你的前后端都启动后，同一个 WiFi 下的其他人应该在浏览器输入：

```text
http://10.90.11.95:5173/
```

不要让别人输入：

```text
http://localhost:5173/
http://127.0.0.1:5173/
```

`localhost` 和 `127.0.0.1` 只代表“当前这台电脑”。别人电脑上输入 localhost，只会访问他们自己的电脑，不会访问你的电脑。

## 二、你截图里的几个地址应该选哪个

你的 Vite 窗口里显示了：

```text
Local:   http://localhost:5173/
Network: http://192.168.64.1:5173/
Network: http://192.168.180.1:5173/
Network: http://10.90.11.95:5173/
```

应该给别人这个：

```text
http://10.90.11.95:5173/
```

原因：

- `localhost`：只给你自己本机访问。
- `192.168.64.1`、`192.168.180.1`：这是 VMware 虚拟网卡地址，一般不是同学手机/电脑所在的 WiFi 网络。
- `10.90.11.95`：这是你当前 WLAN 无线网卡的 IP，别人和你连同一个 WiFi 时应该访问这个。

如果你换了 WiFi，或者断网重连，这个 IP 可能会变。变了以后重新看 Vite 输出里的 `Network` 地址，或者运行：

```powershell
ipconfig
```

找到这一段：

```text
Wireless LAN adapter WLAN:
   IPv4 Address . . . . . . . . . . : xxx.xxx.xxx.xxx
```

然后让别人访问：

```text
http://这里换成你的IPv4地址:5173/
```

## 三、从打开 PowerShell 开始怎么启动

需要打开两个 PowerShell 窗口。

### 第一个 PowerShell：启动后端

进入项目根目录：

```powershell
cd E:\pycharm\CompeteHub
```

启动后端：

```powershell
python scripts\run_backend_dev.py
```

看到类似下面内容，说明后端启动成功：

```text
Running on http://127.0.0.1:5000
```

这个窗口不要关闭。

可以在浏览器打开下面地址测试后端：

```text
http://127.0.0.1:5000/api/v1/health
```

正常会看到类似：

```json
{"code":0,"data":{"service":"competehub-api","status":"ok"},"message":"success"}
```

### 第二个 PowerShell：启动前端

再打开一个新的 PowerShell。

进入项目根目录：

```powershell
cd E:\pycharm\CompeteHub
```

进入前端目录：

```powershell
cd frontend
```

启动前端：

```powershell
"C:\Program Files\nodejs\npm.cmd" run dev
```

看到类似下面内容，说明前端启动成功：

```text
VITE v5.4.21 ready

Local:   http://localhost:5173/
Network: http://10.90.11.95:5173/
```

这个窗口也不要关闭。

## 四、你自己怎么打开

你自己在本机浏览器输入：

```text
http://localhost:5173/
```

或者：

```text
http://10.90.11.95:5173/
```

都可以。

## 五、同 WiFi 的别人怎么打开

让别人先连接和你同一个 WiFi。

然后让他们在浏览器输入：

```text
http://10.90.11.95:5173/
```

如果你的 IP 变了，就把 `10.90.11.95` 换成 Vite 终端里最新显示的 WLAN 地址。

别人也可以先测试接口：

```text
http://10.90.11.95:5173/api/v1/health
```

如果能看到 `status: ok`，说明前端代理到后端也正常。

## 六、常见问题

### 1. 别人打不开页面

先确认：

1. 你的两个 PowerShell 窗口都没有关闭。
2. 你的电脑和对方设备确实连接在同一个 WiFi/局域网。
3. 对方输入的是 `http://10.90.11.95:5173/`，不是 `localhost`。
4. 你的 IP 没有变化。

### 2. 页面能打开，但数据加载失败

检查后端是否还在运行。

你本机打开：

```text
http://127.0.0.1:5000/api/v1/health
```

如果打不开，说明后端停了，需要重新运行：

```powershell
python scripts\run_backend_dev.py
```

### 3. 同 WiFi 也打不开

可能是 Windows 防火墙拦截了端口 `5173`。

可以尝试：

1. Windows 搜索“防火墙”。
2. 打开“允许应用通过 Windows Defender 防火墙”。
3. 找到 Node.js，允许专用网络访问。

如果你会用管理员 PowerShell，也可以添加一条入站规则：

```powershell
New-NetFirewallRule -DisplayName "CompeteHub Frontend 5173" -Direction Inbound -Protocol TCP -LocalPort 5173 -Action Allow
```

### 4. 校园网或公共 WiFi 仍然打不开

有些校园网、酒店 WiFi、公共 WiFi 会开启“AP 隔离”或“客户端隔离”。这种情况下，即使你们连的是同一个 WiFi，也不能互相访问。

解决办法：

- 换一个普通路由器 WiFi。
- 用手机热点，让你和对方都连这个热点。
- 使用 Cloudflare Tunnel/ngrok 这类公网隧道。
- 部署到云服务器。

## 七、公网访问说明

如果希望别人不在同一个 WiFi 下也能访问，需要公网入口。

常见方案：

1. **Cloudflare Tunnel / ngrok**：适合临时演示，会生成一个公网地址。
2. **云服务器**：适合长期运行。
3. **域名 + 服务器 + HTTPS**：适合正式网站。

域名不是当前本地开发必须的。只有你希望别人长期输入一个固定网址，例如：

```text
https://www.competehub.cn
```

才需要购买域名、部署服务器，并配置 HTTPS。

## 八、快速检查清单

演示前按顺序检查：

1. 后端 PowerShell 正在运行。
2. 前端 PowerShell 正在运行。
3. 你自己能打开 `http://localhost:5173/`。
4. 你自己能打开 `http://10.90.11.95:5173/`。
5. 别人和你连接同一个 WiFi。
6. 别人输入 `http://10.90.11.95:5173/`。

当前这次应该给别人的地址就是：

```text
http://10.90.11.95:5173/
```
