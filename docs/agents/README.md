# Agent Guidelines

This directory contains development workflow guidance for coding agents working in this repository.

`AGENTS.md` is the repository-level behavior contract. The documents in this directory provide detailed workflows for specific kinds of work.

## Documents

- `domain.md`: domain documentation layout and consumer rules for engineering skills.
- `feature-prd.md`: Feature PRD drafting rules for agents and local PRD helpers.
- `issue-tracker.md`: GitHub issue tracker conventions used by engineering skills.
- `tdd.md`: test-driven development workflow for bug fixes, behavior changes, and new features.
- `triage-labels.md`: mapping from canonical triage roles to repository labels.

## Local Conventions

- Keep agent workflow documents concise and directly tied to repository commands.
- Update `AGENTS.md` when a workflow becomes a repository-level requirement.
- Update `mkdocs.yml` when adding, removing, or renaming pages in this directory.
