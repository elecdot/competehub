## Notes

- Decision:
  - 每个人在“领取”issues时需要：1. 点击 issue body 右上角 "Assign yourself" 成为任务执行人 2. 留下一个 comment 例如"a", 简单通知一下我。
  - 每个人不止要找到自己的工作，有的时候可能会成为 contributor: `Active` 或是 `Assign with`:
    - Active - 意味着这个 issue 除了 DRI 推进以外，需要多人合作。我们会尽可能避免这种情况.
    - Align with - 意味着该 issue 涉及到了别人的工作，需要你们之间进行对齐。
  - 大家自己需要写好 PR，包括填充 Summary, Notes For Reviewers, 结尾补一个 "Cloese #x"(x 为 issue 编号).
  - 后续工作模型：
    1. 我每天早上发一个这样的工作总结，大家可以找一下自己对应的 issue（这个总结是在咱们的 repo 里的）

````markdown
姓名：
当前 Issue(s)：

完成状态
-

存在问题或解决记录
````
    3. 晚上 10 min 的短会下一步
