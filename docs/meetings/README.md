# Meetings

Meeting notes are lightweight coordination records for the team. They should
help members recover the current owner list and blockers without becoming a
separate project-management system.

GitHub Issues remain the source of truth for implementation work. Meeting notes
may summarize current assignments, blockers, and next actions, but durable scope,
acceptance criteria, validation, and ownership belong in issues and project
documents.

## Daily Standup

Use `daily-standup-template.md` when the team needs a written owner list for the
day.

The owner list should reuse the project Delivery Ownership fields so members and
agents can connect meeting assignments back to GitHub Issues.
