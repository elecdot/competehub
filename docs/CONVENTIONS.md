# File and Naming Conventions

## Filenames

1. Use readable, lowercase English words.
2. Do not rely on case-only differences to distinguish files.
3. Use `_` as the word separator when multiple words are required (for example: `topic_keyword`).
4. Keep names concise and avoid spaces or special characters, especially `:`, `#`, `[`, and `]`.

## Paths

1. Always provide paths relative to the repository root in documentation and instructions.
2. Avoid machine-specific absolute paths to keep content portable across environments.
