# 1．引言

## 1.1 编写目的

本文档说明 CompeteHub 的运行环境、安装初始化方式、主要用户操作流程、常见错误和恢复方法。v0.0 版本作为用户手册初稿，既覆盖当前仓库运行方式，也说明核心功能完成后的预期使用流程。

## 1.2 项目背景

CompeteHub 帮助本科学生查找可信竞赛、判断赛事适配性、收藏或订阅关键节点，并通过个人日历和站内提醒跟进报名、提交和评审等时间点。管理员负责赛事录入、审核、发布和状态维护。

## 1.3 定义

| 术语 | 定义 |
|---|---|
| 学生用户 | 使用系统查找、筛选、订阅赛事的本科学生。 |
| 管理员 | 维护赛事信息、审核发布和管理状态的用户。 |
| 收藏 | 保存赛事，方便后续查看。 |
| 订阅 | 跟进赛事关键时间节点，并生成提醒。 |
| 个人赛事日历 | 按时间展示已订阅赛事节点的页面。 |

# 2．软件概述

## 2.1 目标

系统目标是为学生提供可信、结构化、可筛选、可订阅的竞赛信息入口，减少查找成本和错过关键节点的风险。

## 2.2 功能

学生端功能：

- 查看首页和赛事入口。
- 搜索、筛选、排序赛事。
- 查看赛事详情、来源、时间节点、适配标签和官方链接。
- 收藏赛事。
- 订阅赛事关键节点。
- 查看个人赛事日历。
- 查看站内提醒。
- 维护学生画像和推荐偏好。
- 查看规则推荐和推荐理由。

管理端功能：

- 新建和编辑赛事。
- 保存草稿和提交审核。
- 审核通过或驳回赛事。
- 下架、归档、取消或标记过期赛事。
- 维护基础字典。
- 查看审核记录和操作日志。

## 2.3 性能

预期体验目标：

- 学生在 3 次以内筛选操作中定位一组符合条件的赛事。
- 学生在 2 步以内完成收藏或订阅。
- 列表通过分页加载，避免大量数据一次性渲染。
- 提醒任务异步执行，不阻塞页面操作。

# 3．运行环境

## 3.1 硬件

本地开发和演示建议配置：

- CPU：双核及以上。
- 内存：8 GB 及以上。
- 磁盘：至少 5 GB 可用空间。
- 网络：可访问 npm 和 Python 依赖源；演示时浏览器可访问本地服务。

## 3.2 支持软件

必需软件：

- `just`
- `uv`
- Node.js 和 npm
- Docker Desktop 或兼容 Docker Compose 的环境
- 现代浏览器

运行组件：

- Flask API。
- Vue 3 Web 前端。
- PostgreSQL。
- Redis。
- Celery worker。

# 4. 使用说明

## 4.1 安装和初始化

在仓库根目录执行：

```bash
just api-sync
npm --prefix apps/web install
just infra-up
```

启动后端：

```bash
just api-dev
```

启动前端：

```bash
just web-dev
```

常用验证：

```bash
just api-test
just api-lint
just web-lint
just web-build
just docs-build
docker compose -f infra/docker-compose.yml config
```

停止本地 PostgreSQL 和 Redis：

```bash
just infra-down
```

## 4.2 运行步骤

### 4.2.1 学生查找赛事

1. 打开系统首页。
2. 进入赛事列表。
3. 在搜索框输入赛事名称、主办方或关键词。
4. 按专业、年级、类别、标签、截止时间和参赛形式筛选。
5. 按截止时间、发布时间、推荐度或热度排序。
6. 点击赛事进入详情页。

### 4.2.2 学生查看赛事详情

1. 在详情页查看赛事简介、来源、主办方、类别和官方通道。
2. 查看适合专业、适合年级、参赛形式、团队人数和报名条件。
3. 查看报名截止、作品提交、比赛开始、评审答辩和结果公布等时间节点。
4. 根据来源链接或官方报名链接跳转到外部页面。

### 4.2.3 学生收藏或订阅赛事

1. 登录学生账号。
2. 在赛事列表或详情页点击收藏。
3. 如需跟进提醒，点击订阅。
4. 确认提醒节点和提前提醒天数。
5. 在个人赛事日历或消息中心查看提醒。

### 4.2.4 学生维护画像

1. 登录后进入个人中心。
2. 填写学院、专业、年级、兴趣标签、竞赛经历和目标偏好。
3. 保存后，系统在推荐和筛选默认项中使用最新画像。

### 4.2.5 管理员录入赛事

1. 登录管理员账号。
2. 进入管理端赛事管理。
3. 新建赛事，填写标题、来源、主办方、类别、时间节点、条件、链接和适配信息。
4. 保存为草稿或提交审核。
5. 审核通过后，赛事在学生端可见。

### 4.2.6 管理员审核赛事

1. 进入待审核赛事列表。
2. 查看赛事详情、来源、时间节点、链接和适配标签。
3. 选择通过或驳回。
4. 填写审核意见。
5. 系统记录审核结果和操作日志。

## 4.3 出错和恢复

| 问题 | 可能原因 | 处理方式 |
|---|---|---|
| 后端无法启动 | 依赖未同步或端口占用 | 运行 `just api-sync`，检查端口占用后重启。 |
| 前端无法启动 | npm 依赖缺失 | 运行 `npm --prefix apps/web install`。 |
| 数据库连接失败 | Docker 服务未启动或环境变量错误 | 运行 `just infra-up`，检查 `.env` 或默认配置。 |
| Redis 连接失败 | Redis 容器未运行 | 运行 `just infra-up` 或检查 Docker Compose 状态。 |
| 登录失败 | 账号不存在、密码错误或账号禁用 | 检查账号信息，联系管理员恢复账号。 |
| 搜索无结果 | 关键词或筛选条件过窄 | 清空部分筛选条件或更换关键词。 |
| 不能收藏或订阅 | 未登录或权限不足 | 登录学生账号后重试。 |
| 管理端不可访问 | 当前用户不是管理员 | 使用管理员账号登录。 |
| 提醒未出现 | 赛事无未来节点、订阅已取消或提醒关闭 | 检查赛事时间节点和个人提醒设置。 |
